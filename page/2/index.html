处理 bad interpreter No such file or directory
发表于2018-03-29
执行命令时有时会出现 bad interpreter: No such file or directory 错误，一般是因为Linux无法识别出Windows的DOS格式，此时只需将文件格式转换成Unix的即可，方法如下： 编辑出错文件： vi filename 查看该格式（报错文件格式是DOS）： :set ff 修改格式： :set ff=unix 保存退出： :wq!
